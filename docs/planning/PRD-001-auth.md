# PRD-001: 사용자 인증

## 배경

투자 대시보드는 개인 자산 데이터(포트폴리오, 거래 내역, 수익률 등)를 다루므로 사용자별 데이터 격리가 필수다. 모든 기능의 전제조건으로 인증 시스템을 먼저 구축한다.

## 목표

- 이메일/비밀번호 기반 회원가입 및 로그인
- 이메일 인증 확인 후 가입 완료
- 앱 재시작 후에도 로그인 상태 유지 (자동 로그인)
- 로그아웃

## 범위

### 포함
- 회원가입 (이메일, 비밀번호, 닉네임)
- 이메일 인증 대기 안내 및 인증 완료 처리
- 로그인
- 로그아웃
- 로그인 상태에 따른 화면 분기 (인증 전 → 로그인/회원가입 화면, 인증 후 → 메인 화면)
- 사용자 프로필 정보 저장 (닉네임) — 이메일 인증 완료 후 저장

### 제외
- 소셜 로그인 (추후 추가)
- 비밀번호 찾기 / 재설정 (추후 추가)
- 회원 탈퇴 (추후 추가)

## 사용자 흐름

### 신규 사용자
1. 앱 실행 → 로그인 화면 (기본 탭)
2. "회원가입" 탭 선택
3. 이메일, 비밀번호, 닉네임 입력
4. 유효성 검사 통과 → 가입 요청
5. Supabase가 인증 이메일 자동 발송
6. **이메일 인증 대기 화면** 표시
   - "입력한 이메일로 인증 링크를 보냈습니다. 이메일을 확인해주세요." 안내
   - 인증 메일 재발송 버튼
   - 로그인 화면으로 돌아가기 버튼
7. 사용자가 이메일의 인증 링크 클릭
8. 앱으로 딥링크 복귀 → `SIGNED_IN` 이벤트 감지
9. 프로필(닉네임) 저장 → 메인 화면으로 이동

### 기존 사용자 — 자동 로그인
1. 앱 실행
2. 저장된 세션 확인
   - 유효한 세션 존재 → 메인 화면으로 바로 이동
   - 세션 없음 또는 만료 → 로그인 화면으로 이동

### 기존 사용자 — 수동 로그인
1. 로그인 화면에서 이메일, 비밀번호 입력
2. 로그인 요청
   - 성공 → 메인 화면으로 이동
   - 실패 (이메일 없음 / 비밀번호 불일치) → 에러 메시지 표시
   - 이메일 미인증 상태 → "이메일 인증이 필요합니다. 이메일을 확인해주세요." 표시

### 로그아웃
1. 설정 화면에서 로그아웃 버튼 탭
2. 확인 다이얼로그 표시
3. 확인 → 세션 삭제 → 로그인 화면으로 이동

## 화면 구성

### 로그인/회원가입 화면 (`AuthScreen`)
- 로그인 탭 / 회원가입 탭 전환
- **로그인 탭**
  - 이메일 입력 필드
  - 비밀번호 입력 필드 (마스킹)
  - 로그인 버튼
  - 에러 메시지 영역
- **회원가입 탭**
  - 이메일 입력 필드
  - 비밀번호 입력 필드 (마스킹)
  - 닉네임 입력 필드
  - 회원가입 버튼
  - 에러 메시지 영역
- 로딩 중 버튼 비활성화 및 인디케이터 표시

### 이메일 인증 대기 화면 (`EmailVerificationScreen`)
- 안내 문구: "입력한 이메일로 인증 링크를 보냈습니다."
- 인증 메일 재발송 버튼 (1회 탭 후 60초 쿨다운)
- 로그인 화면으로 돌아가기 버튼

### 설정 화면 (`SettingsScreen`) — 로그아웃 버튼만 포함 (이번 이슈 범위)
- 로그아웃 버튼
- 확인 다이얼로그

## 이메일 인증 기술 처리

### 회원가입 시
- `supabase.auth.signUp()` 호출 → `data.session`이 `null`이면 이메일 인증 대기 상태
- 닉네임은 세션이 없으므로 이 시점에 저장하지 않음
- `EmailVerificationScreen`으로 이동하면서 닉네임을 임시 보관 (state 또는 메모리)

### 인증 링크 클릭 후
- Supabase가 딥링크로 앱 복귀 (`onAuthStateChange` → `SIGNED_IN` 이벤트)
- 이 시점에 세션이 생성되므로 `createProfile(닉네임)` 호출
- 성공 시 메인 화면으로 이동

### 딥링크 설정
- Expo의 `expo-linking`을 통해 Supabase 인증 콜백 URL 처리
- `app.json`에 scheme 설정 필요 (예: `investdashboard://`)
- Supabase 대시보드 → Authentication → URL Configuration에 Redirect URL 등록 필요

## 유효성 검사

| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| 이메일 | 이메일 형식 | 올바른 이메일 형식을 입력해주세요 |
| 이메일 | 중복 불가 (회원가입 시) | 이미 사용 중인 이메일입니다 |
| 비밀번호 | 8자 이상 | 비밀번호는 8자 이상이어야 합니다 |
| 닉네임 | 2자 이상 20자 이하 | 닉네임은 2자 이상 20자 이하로 입력해주세요 |
| 로그인 실패 | — | 이메일 또는 비밀번호가 올바르지 않습니다 |
| 이메일 미인증 | — | 이메일 인증이 필요합니다. 이메일을 확인해주세요 |

## 데이터 모델 (개요)

Supabase Auth가 인증을 담당하므로 `auth.users` 테이블은 Supabase가 관리한다.
앱에서 필요한 추가 프로필 정보(닉네임 등)는 별도 `profiles` 테이블에 저장한다.

| 항목 | 내용 |
|------|------|
| 인증 주체 | Supabase Auth (`auth.users`) |
| 추가 프로필 | `public.profiles` 테이블 (user_id FK → auth.users) |
| 프로필 저장 시점 | 이메일 인증 완료 후 (`SIGNED_IN` 이벤트) |
| 세션 관리 | Supabase Auth 세션 (자동 갱신) |

## 비기능 요구사항

- 로그인 상태는 앱 재시작 후에도 유지 (Supabase 세션 자동 복원)
- 비밀번호 평문 저장 금지 (Supabase Auth에 위임)
- `profiles` 테이블은 본인 데이터만 조회/수정 가능 (RLS)
- 네트워크 오류 시 적절한 에러 메시지 표시
- 다중 기기 로그인 허용 (기존 세션 유지, Supabase Auth 기본 동작)
- 세션 만료 시 자동으로 로그인 화면으로 이동
