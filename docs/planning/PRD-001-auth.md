# PRD-001: 사용자 인증

## 배경

투자 대시보드는 개인 자산 데이터를 다루므로 사용자별 데이터 격리가 필수다.
모든 기능의 전제조건으로 인증 시스템을 먼저 구축한다.

## 목표

- 이메일/비밀번호 기반 회원가입 및 로그인
- 로그인 상태 유지 (자동 로그인)
- 로그아웃

## 범위

### 포함
- 회원가입 (이메일, 비밀번호, 닉네임)
- 로그인
- 로그아웃
- 로그인 상태에 따른 화면 분기 (인증 전 → 로그인 화면, 인증 후 → 메인 화면)

### 제외
- 소셜 로그인 (추후 추가)
- 비밀번호 찾기 (추후 추가)
- 이메일 인증 (추후 추가)

## 사용자 흐름

### 신규 사용자
1. 앱 실행 → 로그인 화면
2. "회원가입" 탭 선택
3. 이메일, 비밀번호, 닉네임 입력
4. 가입 완료 → 메인 화면으로 이동

### 기존 사용자
1. 앱 실행
   - 로그인 상태 유지 중 → 메인 화면으로 바로 이동
   - 세션 만료 또는 최초 실행 → 로그인 화면
2. 이메일, 비밀번호 입력
3. 로그인 완료 → 메인 화면으로 이동

### 로그아웃
1. 설정 또는 프로필 메뉴에서 로그아웃
2. 로그인 화면으로 이동

## 유효성 검사

| 필드 | 규칙 |
|------|------|
| 이메일 | 이메일 형식, 중복 불가 |
| 비밀번호 | 8자 이상 |
| 닉네임 | 2자 이상 20자 이하 |

## 비기능 요구사항

- 로그인 상태는 앱 재시작 후에도 유지
- 비밀번호는 평문 저장 금지 (Supabase Auth 위임)
- 모든 API는 인증된 사용자만 접근 가능 (RLS)
