# PRD-002: 메인 대시보드

## 배경

인증 후 사용자가 처음 보는 화면. 누적 자산 변화와 손익을 한눈에 파악하고, 매매·입출금·배당 같은 계정 이벤트를 기록하거나 CSV로 불러올 수 있어야 한다. 한국 주식 / 미국 주식 / 코인을 통합해서 보여주되 부문별 구분도 유지한다.

핵심 차별 포인트:

- 투자 시작일부터의 **누적 순수익**을 명확히 표시 (오늘 수익률 X)
- **원금 / 예수금 / 수수료 / 세금 / 배당금**을 각각 분리 추적
- 자산 히스토리 그래프 한 장으로 **원금·평가액·예수금**을 동시에 시각화
- 미국 주식·코인은 **달러+원화 동시 표시**

## 목표

- 누적 자산 변화를 단일 그래프로 시각화 (원금·평가액·예수금 + 이벤트 마커)
- 핵심 지표 KPI 카드로 한눈에 표시 (총 평가액, 원금, 순수익, 예수금, 누적 배당/수수료/세금)
- 보유 종목 카드를 한눈에 확인 (실시간 현재가 기준 수익률 포함)
- 계정 이벤트(매수/매도/입금/출금/배당)를 수동 입력 또는 증권사 CSV 업로드로 등록

## 범위

### 포함
- 메인 대시보드 화면 (KPI 카드 → 자산 히스토리 그래프 → 종목 카드 리스트)
- 5종 계정 이벤트 수동 입력 (매수, 매도, 입금, 출금, 배당)
- 증권사 CSV 파일 업로드 및 파싱
- 현재가 실시간 API 연동 (한국 주식 / 미국 주식 / 코인)
- 환율 자동 조회 (1시간 주기 폴링)
- 부문별 필터 (전체 / 한국 주식 / 미국 주식 / 코인)
- 일별 자산 스냅샷 자동 기록

### 제외 (PRD-002a-future로 분리)
- 부문별 자산 비중 파이 차트
- TWR(시간가중수익률) 계산
- 벤치마크(KOSPI / S&P 500) 비교
- 실현/미실현 손익 분리, 환차익 별도 집계
- 외환(KRW↔USD 환전) 이벤트 처리
- 종목별 상세 차트
- 매매 이벤트 달력 뷰 및 메모 기능

## 화면 구성 및 레이아웃

### 메인 대시보드 (`DashboardScreen`)

스크롤 구조. 상단부터 순서대로:

```
[부문 필터 탭] 전체 | 한국주식 | 미국주식 | 코인

[KPI 카드 — 1차 행, 큰 카드 3개]
  총 평가액 │ 원금 │ 순수익 (+수익률%)

[KPI 카드 — 2차 행, 작은 카드 4개]
  예수금 │ 누적 배당금 │ 누적 수수료 │ 누적 세금

[자산 히스토리 그래프]
  - 원금 / 평가액 / 예수금 라인
  - 배당 수령·출금 이벤트 마커
  - 기간 전환: 일 / 주 / 월 / 년 / 전체

[종목 카드 리스트]
  - 부문별 그룹 헤더
  - 각 종목 카드
```

우하단 FAB(+) 버튼 → 계정 이벤트 입력 바텀시트

---

### KPI 카드 섹션

#### 1차 행 (큰 카드 3개)

| 카드 | 값 | 서브 라벨 | 비고 |
|---|---|---|---|
| 총 평가액 | 보유 주식 평가 + 예수금 (KRW 환산) | "원금 + 순수익" | — |
| 원금 | 누적 입금 − 누적 출금 (KRW 환산) | "입금 ₩X − 출금 ₩Y" | — |
| 순수익 | 총 평가액 − 원금 | 수익률(%) | + 초록 / − 빨강 색상 구분 |

#### 2차 행 (작은 카드 4개)

| 카드 | 값 | 서브 라벨 |
|---|---|---|
| 예수금 | 통화별 분리 + KRW 환산 합계 | "₩XXX + $YYY ≈ ₩ZZZ" |
| 누적 배당금 | 모든 배당 이벤트 합 (KRW 환산) | — |
| 누적 수수료 | 모든 거래의 수수료 합 (KRW 환산) | — |
| 누적 세금 | 모든 거래의 세금 합 (KRW 환산) | — |

**부문 필터 연동**: 1차 행은 선택 부문에 한정해 계산. 2차 행 중 예수금은 통화별이므로 필터와 무관 (전체 표시), 배당/수수료/세금은 부문별로 필터.

---

### 자산 히스토리 그래프

단일 차트에 3개 라인 + 2종 이벤트 마커.

#### 라인

| 라인 | 스타일 | 의미 |
|---|---|---|
| 원금 | 회색 점선 + 옅은 회색 영역(바닥부터 채움) | 누적 입금 − 누적 출금 (KRW 환산) |
| 평가액 | 파란 실선 | 보유 주식 평가 + 예수금 (모든 통화 KRW 환산) |
| 예수금 | 진한 회색 실선 (영역 없음) | 보유 현금 (모든 통화 KRW 환산) |

#### 자동 시각 효과

- **원금 ↔ 평가액 사이 영역**: 평가액 > 원금이면 초록(평가익), 평가액 < 원금이면 빨강(평가손). 별도 데이터 없이 두 라인 사이를 채우면 됨.
- **예수금 ↔ 평가액 사이 간격**: 자동으로 "주식에 묶인 금액"을 시각화. 추가 라인·면적 필요 없음.

#### 이벤트 마커

| 이벤트 | 마커 | 위치 |
|---|---|---|
| 배당 수령 | 주황 점 (●) | 해당 시점 평가액 라인 위 |
| 출금 | 회색 역삼각형 (▽) | 해당 시점 원금 라인 위 |

매수/매도 이벤트는 마커로 표시하지 않음 (너무 빈번해서 차트가 지저분해짐). 매수/매도의 효과는 자연스럽게 평가액·예수금 라인의 모양으로 드러난다.

#### 기간 전환

| 단위 | X축 표시 | 데이터 출처 |
|------|---------|------------|
| 일 | 시간 (장 시간) | 실시간 가격 폴링 결과 |
| 주 | 요일 | `daily_snapshots` |
| 월 | 날짜 | `daily_snapshots` |
| 년 | 월 | `daily_snapshots` |
| 전체 | 월 | `daily_snapshots` |

#### 부문 필터 연동

- "전체" → 모든 부문 통합 그래프
- 부문 선택 → 해당 부문의 보유 자산 + 예수금만 반영. 예수금은 부문에 직접 귀속되지 않으므로 "전체" 외에는 예수금 라인 숨김 처리(또는 0으로 표시).

#### 기본 가정

- 신용거래·공매도 미지원 → 예수금 ≤ 총 평가액 항상 성립.
- 모든 통화는 그래프 시점의 환율로 KRW 환산하여 단일 Y축에 표기.

### 종목 카드

카드 1장이 보유 종목 1개를 나타낸다.

**표시 정보**

| 항목 | 내용 |
|------|------|
| 종목명 + 티커 | 예: 삼성전자 005930 |
| 자산 유형 배지 | 한국주식 / 미국주식 / 코인 |
| 보유 수량 | |
| 평균 매수가 | 원화 표시 (미국주식·코인은 원 통화 + 원화 병기) |
| 현재가 | 원 통화 + 원화 병기 (미국주식·코인) |
| 평가금액 | 현재가 × 보유수량 (원화 환산) |
| 수익률 | (현재가 − 평균매수가) / 평균매수가 × 100 (%) |
| 평가손익 | 평가금액 − 투입 원금 (원화, + / − 색상 구분) |

#### CSV 업로드

- 시트 상단 "CSV 파일 업로드" 버튼.
- 증권사별 파일 포맷을 서버(Edge Function)에서 파싱. 토스/삼성/카카오페이증권/업비트의 "전체 거래내역" CSV에는 5종 이벤트가 모두 포함됨.
- 업로드 후 파싱 결과 미리보기 → 확인 후 저장.
- 중복 import 방지: `(user_id, event_date, event_type, ticker, quantity, amount)` 기반 해시로 dedupe.

## 계산 규칙

모든 시간 변수 t는 일자(date) 단위.

### 원금

```
원금(t) = Σ(입금 amount × 입금 시점 환율 KRW 환산)
       − Σ(출금 amount × 출금 시점 환율 KRW 환산)
```

> 입출금 시점의 환율을 고정해 KRW로 환산. 원금이 환율 변동에 흔들리지 않게 함. 환율 변동은 평가액 쪽에서만 반영되어 자연스럽게 환차익/차손으로 흡수.

### 예수금 (통화별)

```
예수금(t, c) = Σ(c 통화 입금) + Σ(c 통화 매도 정산금) + Σ(c 통화 배당)
            − Σ(c 통화 출금) − Σ(c 통화 매수 정산금)
            − Σ(c 통화 수수료) − Σ(c 통화 세금)
```

c ∈ {KRW, USD, ...}.

### 총 평가액

```
총 평가액(t) = Σ_종목i (보유수량_i(t) × 종가_i(t) × 환율_i(t))
            + Σ_통화c (예수금(t, c) × 환율_c(t))
```

> 환율과 종가는 t 시점의 값. 그래프 그릴 때마다 매일 환율을 재반영해 곡선이 환율 영향까지 포함.

### 순수익

```
순수익(t) = 총 평가액(t) − 원금(t)
```

### 수익률 (단순)

```
수익률(t) = 순수익(t) / 원금(t) × 100
```

> 단순 방식. 정확한 TWR은 PRD-002a-future에서 도입. `daily_snapshots`이 있으면 추후 소급 계산 가능.

### 보유수량

매수 누적 − 매도 누적. 단, `corporate_actions` 적용 시 effective_date 이후 보유수량은 비율에 따라 자동 조정 (과거 거래 수량은 그대로 유지).

### 평균 매수가

매수 이벤트 누적 기준 가중평균.

```
평균매수가 = (기존 보유수량 × 기존 평균매수가 + 금번 매수수량 × 금번 체결가)
             ÷ (기존 보유수량 + 금번 매수수량)
```

매도 시 평균매수가는 변경하지 않는다. 전량 매도 후 재매수 시 새로 계산. 기업 액션(분할 등) 발생 시 비율에 따라 조정.

### 환율 처리 정책

- **차트·KPI 표시**: 모든 외화 자산을 t 시점 환율로 KRW 환산 (현재 환율 방식).
- **DB 저장**: 모든 거래는 원 통화 그대로 저장 + `fx_rate_at_event` 함께 기록. 추후 환차익/종목차익 분리(PRD-002a) 시 소급 계산 가능.
- **예수금 KPI 카드**: 통화별로 분리 표시 + 환산 합계 병기. 사용자가 원·달러 보유 현황을 직접 보고 매매 의사결정에 활용.

### 일별 스냅샷 cron

매일 KST 자정 직후 Edge Function이 모든 사용자의 `daily_snapshots` 1행 생성. 코인은 24/7이지만 KST 0시 가격으로 통일.

### 한국 주식 API 전환 계획 (추후)

Twelve Data 무료 티어는 15분 지연 데이터. 실시간 현재가가 필요해지면 **KIS Developers (한국투자증권 Open API)** 로 교체. 계좌 개설 필요. (REST API 기반이므로 OS 중립적, 1년 차 프로모션 수수료율 활용 가능.)

## 비기능 요구사항

- 모든 데이터는 본인 소유만 조회/수정 가능 (RLS).
- 거래 기록은 한 번 저장되면 소급 수정하지 않는다. 정정이 필요하면 별도 정정 이벤트로 처리. 기업 액션은 `corporate_actions` 테이블에만 기록하고 과거 거래는 그대로 둔다.
- CSV 재업로드 시 `(user_id, event_date, event_type, ticker, quantity, amount, external_ref)` 기반 해시로 중복 제거.
- 현재가 API 응답 실패 시 마지막 캐시 값 사용, 갱신 시각 표시.
- 미국 주식·코인 금액의 KRW 환산은 최근 환율 적용, 환율 기준 시각 표시.
- 그래프 데이터 없을 시 빈 상태 안내 문구 표시.
- CSV 업로드 실패 시 실패한 행 번호와 사유 표시.
- 일별 스냅샷 cron 실패 시 다음 실행에서 누락 일자 보충.
- 신용거래·공매도는 지원하지 않음 (예수금 ≤ 총 평가액 가정에 의존).
