# 수동 테스트 회귀 시나리오

전체 기능에 대한 수동 테스트 시나리오. 개발자가 iOS 시뮬레이터(Expo Go)에서 직접 수행한다.

- **수행 시점**: 릴리스 전 전체 회귀, 또는 큰 기능 변경 후 관련 섹션만
- **갱신 규칙**: FE 변경 이슈의 파이프라인이 `09-manual-test.md`를 만들 때 이 문서의 해당 섹션도 함께 갱신한다
- 이슈별 시나리오는 `issues/{NNN}/09-manual-test.md` 참조 (이 문서는 누적 회귀용)

---

## 1. 인증 (PRD-001)

### 1.1 이메일 회원가입 + 이메일 인증
- **사전 조건**: 로그아웃 상태, 미사용 이메일 주소 준비
- **단계**: 회원가입 화면에서 이메일/비밀번호/닉네임 입력 → 가입 → 인증 대기 화면 확인 → 메일의 인증 링크 클릭 → 앱으로 복귀
- **기대 결과**: 인증 대기 화면이 표시되고, 인증 완료 후 가입이 완료되어 메인 화면 진입

### 1.2 로그인 / 로그아웃
- **단계**: 가입된 계정으로 로그인 → 설정 탭 → 로그아웃
- **기대 결과**: 로그인 시 대시보드 진입. 로그아웃 시 인증 화면으로 복귀. 잘못된 비밀번호는 에러 메시지 표시 (앱 크래시 없음)

### 1.3 자동 로그인
- **단계**: 로그인 상태에서 앱 완전 종료 → 재실행
- **기대 결과**: 로그인 화면 없이 바로 대시보드 진입

### 1.4 소셜 로그인 (Google / Apple)
- **단계**: 로그아웃 상태에서 Google 로그인 → 로그아웃 → Apple 로그인
- **기대 결과**: 가입 겸용으로 동작, 닉네임은 provider display name 자동 사용. 동일 이메일 기존 계정이 있으면 자동 연동

## 2. 대시보드 (PRD-002)

### 2.1 초기 로드
- **사전 조건**: 거래 데이터가 있는 계정
- **단계**: 로그인 직후 대시보드 확인
- **기대 결과**: KPI 카드(총 평가액/원금/순수익/예수금/누적 배당·수수료·세금) → 자산 히스토리 그래프(원금·평가액·예수금 3선 + 배당 마커) → 보유 종목 카드 순으로 표시. 미국 주식·코인은 달러+원화 동시 표시

### 2.2 계정 이벤트 수동 입력 (5종)
- **단계**: 이벤트 입력 시트 열기 → 매수/매도/입금/출금/배당 각각 1건씩 등록
- **기대 결과**: 등록 직후 KPI 카드·그래프·종목 카드에 반영. 매수 시 예수금 차감, 입금 시 예수금 증가, 배당 시 누적 배당금 증가

### 2.3 매도 검증
- **단계**: 보유 수량보다 많은 수량으로 매도 시도
- **기대 결과**: 검증 에러 표시, 등록되지 않음

### 2.4 CSV 업로드
- **단계**: CSV 업로드 시트에서 증권사 CSV 선택 → 업로드
- **기대 결과**: 파싱된 이벤트 목록 미리보기 후 일괄 등록, 대시보드 반영

### 2.5 종목 상세
- **단계**: 보유 종목 카드 탭
- **기대 결과**: 종목 상세 시트 표시 — 섹터는 GICS Level 4(Sub-Industry)까지 표시 (PRD-004)

### 2.6 렌더링 회귀
- **단계**: 대시보드에서 새로고침(refetch) 트리거
- **기대 결과**: 기존 콘텐츠가 유지된 채 로딩 표시 (전체 화면 스피너로 대체되지 않음)

### 2.7 당일 스냅샷 수동 새로고침 (일 3회)
- **사전 조건**: 거래 데이터가 있는 계정, 오늘 새로고침 미사용
- **단계**: 그래프 카드 헤더의 `↻ N/3` 버튼 탭 → 호출 중 상태 관찰 → 완료 후 그래프·횟수 확인 → 3회 소진까지 반복
- **기대 결과**:
  - 호출 중 기존 그래프 유지(버튼만 로딩 표시), 완료 후 오늘자 포인트가 현재가 기준 즉시 갱신, 횟수 1 차감
  - 3회 소진 시 버튼 비활성화 + 안내 문구 표시, 앱 재실행에도 유지(서버 강제)
  - 실패(네트워크 오류 등) 시 횟수 미차감, 에러 배너 표시 후 기존 그래프 유지
  - 풀다운 refetch 후에도 서버 데이터가 정상 표시(로컬 새로고침 값이 stale하게 덮어쓰지 않음)
  - 앱 포그라운드 복귀 시 쿼터 재동기화 (KST 자정 경과 시 3/3 리셋)

### 2.8 히스토리 그래프 — 집계 단위·축·마커 (이슈 028)
- **사전 조건**: 거래 데이터(수개월치 스냅샷 + 배당/출금 이벤트)가 있는 계정
- **단계**: 단위 버튼(일/주/월/년) 순차 전환 → 가로 스크롤로 과거 탐색 → X/Y축 라벨·눈금선 확인 → 마커 확인
- **기대 결과**:
  - 단위 버튼은 일/주/월/년 4개 ("전체" 없음). 전환은 버킷 크기만 바꾸며 서버 재조회 없이 즉시 반영, 어느 단위든 전체 히스토리 표시
  - 버킷 값 = 버킷 내 마지막 스냅샷 값. 초기 스크롤 위치는 최신 날짜(오른쪽 끝)
  - X축 라벨: 일별·주별 `MM.DD`, 월별 `YY.MM`, 연별 `YYYY` — 겹침 없음
  - Y축: 만/억 축약 금액 라벨 + 옅은 가로 눈금선, 3선 통합 min~max 스케일
  - 원금 음수 구간은 0 기준선(대시선) 아래로 클리핑 없이 렌더링
  - 이벤트 마커는 배당(●)만 — 출금 마커(▽) 없음 (범례 포함)
  - 스냅샷 0개/1개 계정에서도 크래시 없음

## 3. 메모 (PRD-003)

### 3.1 메모 작성 (엔티티 연결)
- **단계**: 메모 탭 → 새 메모 → 본문 작성 → 종목 연결(검색 모달에서 종목명/티커 검색) → 섹터 연결 → 저장
- **기대 결과**: 목록 최상단에 표시, 연결 엔티티가 칩으로 표시 (종목 칩은 종목명 라벨)

### 3.2 목표가 메모
- **단계**: 종목 연결 메모에 목표가(goal_price) 입력 후 저장 → 재편집
- **기대 결과**: 목표가 저장/표시/수정 정상

### 3.3 메모 수정 / 삭제
- **단계**: 기존 메모 열기 → 본문·연결 엔티티 수정 → 저장 → 다른 메모 삭제
- **기대 결과**: 수정 반영, 삭제 시 목록에서 제거 (메모는 자유 수정·삭제 가능이 원칙)

### 3.5 메모 측 매매이벤트 연결 (PRD-003 §6.4)
- **사전 조건**: 본인 계정에 buy/sell 매매이벤트가 여러 건 존재 (입금/출금/배당도 일부 섞여 있으면 좋음)
- **단계**: 메모 작성 화면 → "매매이벤트" 행 탭 → 선택 모달에서 (a) 목록이 buy/sell만 최신순으로 표시되는지 (b) 종목명·티커 검색(디바운스) (c) 기간 필터 (d) 복수 선택 → 완료 → 칩 누적 표시 → 저장. 이후 같은 메모 재편집 → 모달에서 기존 연결 확인 → 일부 해제 + 신규 추가 → 저장
- **기대 결과**:
  - 후보는 buy/sell만 노출, 입금/출금/배당 제외. 항목은 `발생일 · 매수/매도 · 종목명 · 수량 @ 체결가(통화)` 형식, 매수/매도 색상·라벨 구분
  - 종목이 연결된 메모는 그 종목명이 검색어로 시드되어 관련 이벤트가 먼저 보임. 검색어를 지우면 전체 목록 탐색 가능 (종목 미연결 메모도 동일)
  - 검색/기간 필터 결과도 최신순 유지, 필터 변경 시 기존 목록이 사라지지 않음 (렌더링 원칙)
  - 복수 선택 칩 누적, 같은 이벤트 중복 선택 차단
  - 편집 시 모달에서 추가·해제한 결과가 저장에 실제 반영됨 (이전엔 "유지만" 하던 버그 수정). 저장 후 재편집 시 변경된 연결이 유지됨

### 3.4 메모 필터
- **단계**: 기간 필터 → 종목 필터(복수 선택) → 섹터 필터(복수) → 카테고리 필터 → "연결 없음" 필터를 각각 적용/해제
- **기대 결과**:
  - 같은 타입 내 복수 선택은 OR, 타입 간은 AND
  - 종목 필터는 직접 연결 + 매매이벤트 경유 연결 메모를 함께 반환
  - "연결 없음"은 다른 필터를 무시하고 연결 없는 메모만 표시
  - 필터 변경 시 기존 목록이 사라지지 않고 갱신 (렌더링 원칙)

## 4. 섹터 (PRD-004, PRD-005)

### 4.1 계층 선택
- **단계**: 메모 편집 → 섹터 picker → L1 펼침 → L2 선택
- **기대 결과**: 메모 필터·연결은 Level 2(Industry Group)까지 표시, 계층 펼침/선택/해제 정상

### 4.2 섹터 키워드 검색
- **단계**: 섹터 picker 상단 검색 필드에 키워드 입력 (예: "전기")
- **기대 결과**: 매칭 섹터가 전체 계층 경로(breadcrumb, L1→L4)와 함께 즉시 표시 (클라이언트 검색 — 입력마다 서버 호출 없음). 결과 탭 시 선택/해제 동작

### 4.3 섹터 필터 자기 포함
- **단계**: 메모에 L2 섹터 연결 → 메모 필터에서 같은 L2 선택
- **기대 결과**: 해당 메모와 하위 레벨 연결 메모가 모두 조회됨

## 5. 사용자 카테고리 (PRD-006)

### 5.1 카테고리 CRUD
- **단계**: 설정 탭 → 카테고리 관리 → 생성("2차전지") → 이름 변경 → 삭제
- **기대 결과**: 생성/수정/삭제 즉시 반영. 삭제 시 연결된 메모·종목 매핑 처리가 기획대로 동작

### 5.2 카테고리-종목 배정
- **단계**: 카테고리 선택 → 종목 추가(검색) → 종목 제거
- **기대 결과**: N:M 배정 정상, 같은 종목을 여러 카테고리에 배정 가능

### 5.3 메모-카테고리 연결 + 필터
- **단계**: 메모 편집에서 카테고리 연결 → 저장 → 메모 필터에서 해당 카테고리 선택
- **기대 결과**: 연결 칩 표시, 필터 시 해당 메모 조회. 메모 탭 경유 카테고리 관리 화면 진입/복귀 정상 (설정 탭 경유와 동일 동작)

## 6. 공통 회귀

- 탭 전환(대시보드↔메모↔설정) 시 각 탭 상태 유지, 화면 깜빡임 없음
- 다른 계정으로 로그인 시 이전 계정 데이터가 보이지 않음 (RLS)
- 네트워크 오류 시 에러 표시 후 재시도 가능 (크래시 없음)
