# [20260615-market-price-loading-fe] 수동 테스트 시나리오

> 개발자가 iOS 시뮬레이터(Expo Go)에서 직접 수행한다. 완료 후 issue.md의 `- [ ] 수동 테스트`를 체크한다.
> 변경 범위: 대시보드 현재가 2단계 로딩(저장 종가 baseline 즉시 표시 → 실시간 점진 갱신), Twelve Data 분당 8크레딧 청크 처리, 실패 종목 baseline 유지, 갱신 인디케이터.

## 테스트 데이터 준비 메모
- 미국주식 **9개 이상** 보유한 계정이 핵심 케이스(8개 청크 경계 + 60초 대기 검증). 가능하면 9~16개 보유 계정을 준비한다.
- 미국주식 8개 이하, 한국주식·코인 혼합 계정도 별도로 준비(단일 청크 즉시 완료 케이스).
- `prices` 테이블에 해당 종목들의 EOD 종가가 들어있어야 baseline이 보인다(없으면 일별 종가 cron 또는 시드 확인).

---

## 시나리오 1 — baseline 즉시 표시 (전체 스피너 제거)

- **사전 조건**: 거래 데이터가 있는 계정, `prices`에 보유 종목 EOD 종가 존재
- **단계**:
  1. 로그인 직후 대시보드 진입을 관찰한다(가능하면 화면 녹화).
  2. 진입 순간부터 카드/KPI가 처음 채워지기까지의 시간을 본다.
- **기대 결과**:
  - 전체 화면 스피너가 길게 머무르지 않고, 저장 종가(EOD) 기준으로 KPI·보유 종목 카드가 **즉시** 채워진다.
  - baseline 가격 카드에는 "저장 종가 기준 (M월 D일) — 실시간 갱신 중" 노트가 보인다.
  - 헤더 하단에 "현재가 갱신 중..." 인디케이터(ActivityIndicator)가 표시된다.

## 시나리오 2 — 실시간 점진 갱신 (미국주식 8개 이하 / 단일 청크)

- **사전 조건**: 미국주식 8개 이하 + 한국주식·코인 혼합 보유 계정
- **단계**:
  1. 대시보드 진입 후 baseline 표시를 확인한다.
  2. 잠시(수 초) 기다리며 카드 가격 변화를 관찰한다.
- **기대 결과**:
  - 60초 대기 없이 한 번에(첫 청크) 한국주식·코인·미국주식 현재가가 실시간 값으로 갱신된다.
  - 실시간으로 갱신된 카드는 "저장 종가 기준..." 노트가 사라진다.
  - 모든 갱신 완료 후 "현재가 갱신 중..." 인디케이터가 사라진다.

## 시나리오 3 — 청크 분할 + 60초 대기 (미국주식 9개 이상)

- **사전 조건**: 미국주식 **9개 이상** 보유 계정
- **단계**:
  1. 대시보드 진입 → baseline 즉시 표시 확인.
  2. 첫 청크(미국주식 8개 + 한국주식·코인 전체)가 먼저 실시간 갱신되는지 본다.
  3. 9번째 이후 미국주식 카드는 baseline을 유지한 채, 약 **60초 후** 다음 청크가 갱신되는지 관찰한다(인디케이터 유지 확인).
- **기대 결과**:
  - 첫 8개 미국주식 + 한국주식·코인은 빠르게 실시간 갱신, 나머지 미국주식은 baseline("저장 종가 기준...") 유지.
  - 약 60초 후 다음 8개 미국주식이 실시간 값으로 갱신된다.
  - 잔여분이 모두 갱신될 때까지 "현재가 갱신 중..." 인디케이터가 유지되고, 전부 완료되면 사라진다.
  - 대기 중에도 화면은 채워진 상태이며 빈칸·스피너 전환이 없다.

## 시나리오 4 — 실시간 조회 실패 시 baseline 유지

- **사전 조건**: (가능하면) 일부 종목 실시간 조회 실패를 유도(예: 네트워크 일시 차단 또는 잘못된 종목). 어려우면 정상 상태에서 카드가 빈칸이 되지 않는지 확인하는 것으로 대체.
- **단계**:
  1. 대시보드 진입 → baseline 표시.
  2. 실시간 갱신 진행 중 일부 종목이 실패하는 상황을 관찰.
- **기대 결과**:
  - 실패 종목 카드는 빈칸/0이 되지 않고 저장 종가(baseline) 값과 노트를 그대로 유지한다.
  - 다른 종목 갱신은 정상 진행되며 앱 크래시가 없다.

## 시나리오 5 — 새로고침(refetch) 동작 + 진행 중 청크 취소

- **사전 조건**: 미국주식 9개 이상 보유 계정(60초 대기 구간 존재)
- **단계**:
  1. 대시보드 진입 후 첫 청크 갱신을 확인한다.
  2. 60초 대기/다음 청크 진행 중에 **당겨서 새로고침(refetch)** 을 트리거한다.
  3. 갱신 동작을 관찰한다.
- **기대 결과**:
  - 새로고침 시 기존 콘텐츠가 유지된 채(전체 스피너로 대체되지 않음) baseline → 실시간 갱신 사이클이 다시 시작된다.
  - 이전 청크 루프가 취소되어 가격이 뒤섞이거나 중복 갱신되지 않는다.
  - 콘솔/앱에 "setState on unmounted" 경고나 크래시가 없다.

## 시나리오 6 — 화면 이탈/언마운트 중 cleanup

- **사전 조건**: 미국주식 9개 이상 보유 계정
- **단계**:
  1. 대시보드 진입 후 60초 대기 구간(인디케이터 표시 중)에 진입한다.
  2. 대기 중에 다른 탭(메모/설정)으로 전환하거나 종목 상세 시트를 여닫는다.
  3. 다시 대시보드로 돌아온다.
- **기대 결과**:
  - 탭 전환 시 진행 중 청크 루프가 정리되어, 백그라운드에서 setState 경고나 크래시가 발생하지 않는다.
  - 대시보드 복귀 시 상태가 정상적으로 표시된다.

## 시나리오 7 — 인접 기능 회귀

- **단계**:
  1. KPI 카드·자산 히스토리 그래프·보유 종목 카드가 정상 렌더링되는지 확인(2.1 초기 로드).
  2. 종목 상세 시트(2.5), 당일 스냅샷 수동 새로고침 `↻ N/3`(2.7)이 정상 동작하는지 확인.
  3. 미국주식·코인의 달러+원화 동시 표시가 유지되는지 확인.
- **기대 결과**: 기존 대시보드 기능이 현재가 로딩 변경의 영향 없이 정상 동작한다.
