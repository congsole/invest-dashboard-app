# [028] 수동 테스트 시나리오 — 히스토리 그래프 개편

iOS 시뮬레이터(Expo Go)에서 개발자가 직접 수행한다. 완료 후 issue.md의 `- [ ] 수동 테스트`를 체크한다.

공통 사전 조건: 거래 데이터(스냅샷 수개월치 + 배당/출금 이벤트 각 1건 이상)가 있는 계정으로 로그인.

---

## 1. 집계 단위 전환 (서버 재조회 없음)

- **사전 조건**: 대시보드 진입, 히스토리 그래프 표시 상태
- **단계**:
  1. 그래프의 단위 버튼이 `일 / 주 / 월 / 년` 4개뿐인지 확인 ("전체" 버튼 없음)
  2. 일 → 주 → 월 → 년 순서로 전환
  3. 전환하는 동안 네트워크/로딩 인디케이터 관찰
- **기대 결과**:
  - 어느 단위에서든 전체 히스토리가 그려진다 (표시 범위가 줄어들지 않음)
  - 단위 전환이 즉시 반영되고 서버 재조회(로딩 스피너·콘텐츠 깜빡임)가 없다
  - 그래프 외 영역(KPI 카드, 보유 종목)은 리렌더링되지 않는다 (깜빡임 없음)

## 2. 버킷 값 = 버킷 내 마지막 스냅샷

- **사전 조건**: 특정 주(週)에 스냅샷이 여러 개 있는 데이터
- **단계**: 일별에서 해당 주의 마지막 날짜 값을 확인 → 주별로 전환하여 해당 주 버킷 값 비교
- **기대 결과**: 주별 버킷 값이 그 주의 **마지막 스냅샷 값**과 일치 (평균/합계 아님). 월별·연별도 동일 원리

## 3. 가로 스크롤 탐색

- **단계**:
  1. 일별 단위에서 그래프 초기 표시 위치 확인
  2. 왼쪽으로 스크롤하여 과거 데이터 탐색 → 다시 오른쪽 끝으로
  3. 주별/월별로 전환 후에도 스크롤 동작 확인
- **기대 결과**:
  - 초기 위치는 최신 날짜(오른쪽 끝)
  - 전체 기간을 가로 스크롤로 탐색 가능, 스크롤이 끊기거나 라인이 어긋나지 않음

## 4. X축 라벨

- **단계**: 일별 → 주별 → 월별 → 연별 각각에서 X축 라벨 형식과 간격 확인
- **기대 결과**:
  - 일별·주별 `MM.DD`, 월별 `YY.MM`, 연별 `YYYY`
  - 라벨끼리 겹치지 않음 (자동 간격 배치), 가장 오른쪽(최신) 라벨은 항상 표시
  - 주별 라벨 날짜가 해당 주 월요일 기준으로 표시 (KST 기준 하루 밀림 없음 — 리뷰에서 수정된 타임존 버그 회귀 확인)

## 5. Y축 금액 라벨 + 눈금선

- **단계**: 그래프 좌측 Y축 영역과 가로 눈금선 확인
- **기대 결과**:
  - Y축에 금액 라벨 5개가 만/억 단위 축약으로 표시 (예: `1.2억`, `3,500만`)
  - 옅은 가로 눈금선이 라벨 위치와 일치
  - Y축 스케일은 평가액·원금·예수금 3선의 통합 최소~최대를 모두 포함 (어떤 라인도 위/아래로 잘리지 않음)

## 6. 원금 음수 구간 렌더링

- **사전 조건**: 누적 출금 > 누적 입금 구간이 있는 계정 (없으면 테스트 데이터로 출금 이벤트를 크게 등록)
- **단계**: 해당 구간이 보이도록 스크롤/단위 전환
- **기대 결과**: 원금 라인이 0 기준선 **아래로 잘리지 않고** 렌더링되고, 0 기준선이 대시선으로 표시됨

## 7. 이벤트 마커 — 배당만

- **사전 조건**: 배당 이벤트와 출금 이벤트가 모두 등록된 계정
- **단계**:
  1. 그래프에서 마커 확인
  2. 범례 확인
  3. 주별/월별로 전환하여 마커 위치 확인
- **기대 결과**:
  - 배당 마커(●)만 표시되고 출금 마커(▽)는 어디에도 없음 (범례 포함)
  - 단위 전환 시 마커가 배당 날짜가 속한 버킷 위치에 정확히 표시됨

## 8. 인접 기능 회귀

### 8.1 당일 스냅샷 수동 새로고침 (이슈 026/027)
- **단계**: 그래프 카드 헤더의 `↻ N/3` 버튼 탭
- **기대 결과**: 호출 중 기존 그래프 유지(버튼만 로딩), 완료 후 오늘자 포인트 갱신·횟수 차감 — 그래프 개편 후에도 동일 동작

### 8.2 대시보드 초기 로드 / refetch
- **단계**: 앱 재실행으로 초기 로드 → 풀다운 refetch
- **기대 결과**: 초기 로드에서 그래프 정상 표시. refetch 시 기존 콘텐츠 유지된 채 갱신 (전체 화면 스피너 없음)

### 8.3 부문 필터 전환
- **단계**: 대시보드 부문 필터(전체/한국주식/미국주식 등) 전환
- **기대 결과**: 필터 전환 시 히스토리 그래프 서버 재조회 없음, 화면 깜빡임 없음

### 8.4 이벤트 입력 후 반영
- **단계**: 배당 이벤트 1건 신규 등록 → 대시보드 복귀/refetch
- **기대 결과**: 그래프에 새 배당 마커 반영, KPI 누적 배당금 증가

## 9. 엣지 케이스

- **신규 계정(스냅샷 0개)**: 그래프 영역이 크래시 없이 빈 상태/안내 표시
- **스냅샷 1개**: 단일 포인트가 정상 표시 (라인/스케일 크래시 없음)
- **값 변동 없는 구간(min == max)**: Y축 스케일이 0 분모 없이 정상 표시
