# [028] 히스토리 그래프 개편 — 집계 단위 전환, 축 표시, 배당 마커만

## 개요

자산 히스토리 그래프의 기간 전환 개념을 "조회 범위"에서 "집계 단위"로 재정의하고, 배당 이벤트 마커만 유지하며, Y축 금액 라벨·눈금선을 추가하는 개편이다.

주요 변경:
- **집계 단위 전환**: 버튼(일별/주별/월별/연별)은 표시 범위가 아닌 버킷 크기를 바꾼다. 어느 단위에서든 전체 히스토리를 그리고 가로 스크롤로 탐색. "전체" 버튼 제거.
- **클라이언트 버킷팅**: `daily_snapshots` 전체를 1회 조회하고 버킷팅은 클라이언트에서 수행 (단위 전환 시 서버 재조회 없음). 버킷 값 = 버킷 내 마지막 스냅샷 값.
- **이벤트 마커 축소**: 출금 마커(▽) 제거, 배당 마커(●)만 유지. `get_history_markers` RPC 수정 필요 (파라미터 제거 + `'withdraw'` 필터 제거).
- **Y축 표시**: 금액 라벨 추가 (만/억 단위 축약), 옅은 가로 눈금선. Y축 스케일은 전체 라인 통합 최소~최대 기준. 원금 음수 구간도 0 기준선 아래로 렌더링.

BE 트랙: `get_history_markers` RPC 시그니처 변경 (p_from/p_to 파라미터 제거, `'withdraw'` 필터 제거 → 배당만 반환).
FE 트랙: 클라이언트 버킷팅 로직, 가로 스크롤, X축 라벨 형식, Y축 표시, 출금 마커 제거.
