# [026] 수동 테스트 시나리오 — 메모 측 매매이벤트 연결 UI

개발자가 iOS 시뮬레이터(Expo Go)에서 직접 수행한다. 각 시나리오는 **사전 조건 / 단계 / 기대 결과**로 구성된다.

> 변경 범위: 메모 작성/편집 화면에 "매매이벤트" 선택 행 + 선택 모달 추가 (PRD-003 §6.4). 데이터 레이어는 기존 구현 재사용.

---

## 사전 준비 (공통 사전 조건)

- 로그인 상태.
- 본인 계정에 **매수(buy)·매도(sell) 이벤트가 여러 건** 존재해야 한다. 여러 종목(예: 삼성전자, Apple 등 국내+해외)에 걸쳐 있고, 날짜가 분산되어 있으면 좋다.
- **입금/출금/배당 이벤트도 일부** 등록해 둔다 (후보에서 제외되는지 확인용).
- 가능하면 매매이벤트가 20건 이상이면 검색/기간 필터 효과를 제대로 확인할 수 있다.

---

## 1. 매매이벤트 선택 행 노출 및 후보 범위

- **사전 조건**: 위 공통 사전 조건.
- **단계**:
  1. 메모 탭 → 새 메모 작성 화면 진입.
  2. 종목/섹터 연결 행 사이에 "매매이벤트" 연결 행이 있는지 확인.
  3. 행을 탭해 선택 모달을 연다.
- **기대 결과**:
  - "매매이벤트" 행이 기존 종목/섹터 연결 행과 동일한 UX로 표시된다.
  - 모달에 본인의 **buy/sell 이벤트만** 최신순(`event_date desc`)으로 나열된다.
  - **입금/출금/배당 이벤트는 목록에 나타나지 않는다.**
  - 각 항목이 `발생일 · 매수/매도 · 종목명 · 수량 @ 체결가(통화)` 형식으로 표시되고, 매수와 매도가 색상·라벨로 구분된다.

## 2. 종목명/티커 검색 (디바운스)

- **사전 조건**: 모달이 열린 상태, 여러 종목의 매매이벤트 존재.
- **단계**:
  1. 모달 상단 검색 필드에 종목명 일부 입력 (예: "삼성").
  2. 티커로도 검색 (예: "AAPL").
  3. 검색어를 모두 지운다.
- **기대 결과**:
  - 입력 후 약 300ms 뒤 결과가 갱신된다 (타이핑마다 즉시 호출되지 않음).
  - 매칭되는 종목의 이벤트만 최신순으로 표시된다.
  - 검색 중에도 기존 목록이 사라져 빈 화면이 되지 않는다 (로딩 인디케이터/기존 목록 유지).
  - 검색어를 지우면 전체 buy/sell 목록이 다시 최신순으로 표시된다.

## 3. 기간 필터

- **사전 조건**: 모달이 열린 상태, 날짜가 분산된 매매이벤트 존재.
- **단계**:
  1. 시작일~종료일 범위를 지정한다.
  2. 범위를 변경하거나 해제한다.
- **기대 결과**:
  - 지정 범위 내 이벤트만 표시되고, 최신순 정렬이 유지된다.
  - 기간 필터 변경 시 서버 재호출 없이 즉시 반영되며 기존 목록이 깜빡이지 않는다 (클라이언트 필터).
  - 검색 + 기간 필터를 함께 적용해도 두 조건이 모두 반영된다.

## 4. 복수 선택 / 중복 차단 / 칩 표시

- **사전 조건**: 모달이 열린 상태.
- **단계**:
  1. 여러 이벤트를 선택한다.
  2. 같은 이벤트를 다시 탭해 선택 해제 후 재선택해 본다.
  3. 완료를 눌러 모달을 닫는다.
- **기대 결과**:
  - 선택한 이벤트가 칩으로 누적 표시된다.
  - 같은 이벤트가 중복으로 두 번 추가되지 않는다.
  - 완료 후 메모 편집 화면의 "매매이벤트" 섹션에 선택된 이벤트 칩 목록이 표시된다.

## 5. 선택 시드 (연결 종목 기반)

- **사전 조건**: 매매이벤트가 있는 종목을 메모에 먼저 연결.
- **단계**:
  1. 메모 작성 화면에서 종목을 먼저 연결한다 (해당 종목에 매매이벤트가 있어야 함).
  2. "매매이벤트" 행을 탭해 모달을 연다.
  3. 검색어를 지운다.
- **기대 결과**:
  - 모달이 열릴 때 연결된 종목명이 검색어로 **시드**되어 관련 이벤트가 첫 화면에 바로 보인다.
  - 검색어를 지우면 시드 제약 없이 전체 매매이벤트 목록을 탐색할 수 있다.
  - 종목 미연결 메모에서 모달을 열면 처음부터 전체 목록이 보인다.

## 6. 작성 저장 → 연결 영속

- **사전 조건**: 매매이벤트를 선택한 새 메모.
- **단계**:
  1. 본문 입력 + 매매이벤트 복수 선택 후 저장.
  2. 목록에서 해당 메모를 다시 연다 / 상세 확인.
- **기대 결과**:
  - 저장이 성공하고, 연결된 매매이벤트가 칩(초록, §5) 및 상세 화면에 표시된다.
  - 재진입 시 선택했던 매매이벤트 연결이 그대로 유지된다.

## 7. 편집 시 추가/해제 반영 (핵심 — 기존 버그 수정)

- **사전 조건**: 매매이벤트가 이미 연결된 기존 메모.
- **단계**:
  1. 해당 메모를 편집 모드로 연다.
  2. "매매이벤트" 모달을 열어 기존 연결이 선택 상태로 표시되는지 확인.
  3. 기존 연결 일부를 **해제**하고, 새 이벤트를 **추가**한다.
  4. 저장한다.
  5. 메모를 다시 열어 연결 상태를 확인한다.
- **기대 결과**:
  - 편집 모달에 기존에 연결된 이벤트가 선택된 상태로 표시된다.
  - 해제/추가한 결과가 **저장 후 실제로 반영**된다 (이전에는 편집 화면이 연결을 "유지만" 하고 변경하지 못하던 문제가 해결됨).
  - 재진입 시 변경된 연결 상태가 유지된다.

---

## 8. 인접 기능 회귀

- **메모 종목/섹터 연결**: 매매이벤트 섹션 추가로 인해 기존 종목·섹터·카테고리 연결 행이 정상 동작하는지 확인 (선택/해제/저장).
- **목표가 입력**: 종목 연결 시 목표가 입력 필드가 여전히 정상 노출/저장.
- **메모 필터 — 종목**: 메모 필터에서 종목 선택 시 직접 연결 + **매매이벤트 경유 연결** 메모가 함께 반환되는지 확인.
- **메모 작성/수정/삭제 일반 흐름**: 본문만 있는 메모, 연결 없는 메모 저장 정상. 삭제 정상.
- **렌더링**: 모달 열림/검색/필터 시 화면 깜빡임 없음, 메모 목록 스크롤 상태 유지.
- **RLS**: 다른 계정으로 로그인 시 본인 매매이벤트만 모달에 노출.
