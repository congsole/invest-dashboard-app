# [027] 당일 스냅샷 수동 새로고침 — 수동 테스트 시나리오

iOS 시뮬레이터(Expo Go)에서 개발자가 직접 수행한다. 완료 시 issue.md의 `- [ ] 수동 테스트`를 체크한다.

> 참고: 일 3회 제한은 서버(`snapshot_refresh_quotas`)에서 강제되므로, 횟수 소진 테스트 후 재테스트하려면
> SQL Editor가 아닌 로컬 Supabase 또는 다른 테스트 계정을 사용한다 (운영 DB 직접 수정 금지).

---

## 1. 새로고침 버튼 표시 및 초기 쿼터 동기화

- **사전 조건**: 거래 데이터(account_events)가 있는 계정으로 로그인, 오늘 새로고침 미사용 상태
- **단계**:
  1. 대시보드 진입
  2. 자산 히스토리 그래프 카드 헤더 확인
- **기대 결과**:
  - 제목("자산 히스토리") 오른쪽에 `↻ 3/3` pill 버튼이 표시된다
  - 버튼은 활성 상태(탭 가능)이며 안내 배너는 보이지 않는다

## 2. 새로고침 성공 — 그래프 즉시 갱신 + 횟수 차감

- **사전 조건**: 시나리오 1 직후 상태. 가능하면 오늘 매수/입금 이벤트를 1건 등록해 둔다 (당일 스냅샷 변화 확인용)
- **단계**:
  1. `↻ 3/3` 버튼 탭
  2. 호출 중 그래프 카드 상태 관찰
  3. 완료 후 그래프와 버튼 표시 확인
- **기대 결과**:
  - 호출 중: 버튼 아이콘 자리가 ActivityIndicator로 교체되고 버튼은 비활성화되지만, **기존 그래프는 사라지지 않고 유지**된다 (전체 화면 스피너 금지 — 렌더링 원칙)
  - 완료 후: 그래프의 오늘자 데이터 포인트가 현재가 기준으로 즉시 갱신된다 (당일 이벤트 반영)
  - 버튼이 `↻ 2/3`으로 갱신된다

## 3. 중복 탭 방지

- **단계**: 새로고침 버튼을 빠르게 연속 두 번 탭
- **기대 결과**: 첫 호출 진행 중에는 버튼이 비활성화되어 두 번째 탭이 무시된다. 횟수는 1회만 차감된다

## 4. 횟수 소진 — 버튼 비활성화 + 안내 문구

- **사전 조건**: 남은 횟수 2회 상태 (시나리오 2 완료 후)
- **단계**:
  1. 새로고침을 2회 더 수행하여 `↻ 0/3` 도달
  2. 버튼과 카드 하단 확인
  3. 버튼 탭 시도
- **기대 결과**:
  - 버튼이 비활성화(흐린 색상)되고 탭해도 반응 없음
  - 카드 하단에 안내 배너 표시: "오늘 새로고침 횟수를 모두 사용했습니다. 내일 다시 이용할 수 있습니다."
  - 앱을 종료 후 재실행해도 `0/3` 상태가 유지된다 (서버 쿼터 동기화)

## 5. 포그라운드 복귀 시 쿼터 재동기화

- **사전 조건**: 새로고침을 1회 이상 사용한 상태
- **단계**:
  1. 홈 화면으로 나가 앱을 백그라운드로 전환
  2. 수 초 후 앱으로 복귀
- **기대 결과**: 그래프 카드의 `↻ N/3` 표시가 서버 쿼터 기준으로 재동기화된다 (KST 자정이 지나 복귀한 경우 `3/3`으로 리셋)

## 6. 새로고침 에러 처리

- **사전 조건**: 남은 횟수 1회 이상
- **단계**:
  1. 기기 네트워크 차단(시뮬레이터: macOS Wi-Fi 끔) 후 새로고침 버튼 탭
  2. 에러 배너 확인 후 닫기 버튼 탭
  3. 네트워크 복구 후 다시 새로고침
- **기대 결과**:
  - 사용자 친화적 에러 메시지 배너가 표시되고, 기존 그래프는 유지된다
  - 에러 배너는 기존 데이터 로드 에러 배너와 독립적으로 표시/닫기 가능
  - 실패한 시도는 횟수가 차감되지 않는다 (복구 후 동일 `N/3` 확인)
  - 복구 후 새로고침은 정상 동작한다

## 7. 풀다운 새로고침(refetch)과의 상호작용 (회귀)

- **사전 조건**: 수동 새로고침을 1회 수행한 직후
- **단계**:
  1. 대시보드를 풀다운하여 전체 refetch
  2. 그래프 확인
- **기대 결과**: refetch된 서버 데이터가 그대로 표시된다 (이전 수동 새로고침의 로컬 값이 오늘 날짜를 stale하게 덮어쓰지 않음). 쿼터 표시는 유지된다

## 8. 인접 기능 회귀

- **단계**:
  1. 그래프 기간 탭(기존 periodSelector) 전환
  2. 이벤트 입력 시트에서 매수 1건 등록
  3. KPI 카드·보유 종목 카드 확인
  4. 탭 전환(대시보드↔메모↔설정) 후 복귀
- **기대 결과**:
  - 기간 탭 전환이 기존과 동일하게 동작한다 (헤더 레이아웃 변경에 의한 회귀 없음)
  - 이벤트 등록 후 대시보드가 정상 갱신된다 (KPI/그래프/종목 카드)
  - 탭 전환 후 복귀 시 쿼터 표시와 그래프 상태가 정상이다
